⚠️ Critical Findings

🔴 Secrets Checked Into Repository CRITICAL

The .env file contains VITE_SUPABASE_* values and is committed to git. .gitignore does not exclude .env. This exposes project-specific API keys and must be remedied immediately.

🔴 JWT Verification Disabled CRITICAL

supabase/config.toml sets verify_jwt = false for many functions. In production, JWTs should be verified for functions requiring authenticated access.

🟠 Missing Tests & CI HIGH

No unit/e2e tests detected (Vitest/Jest/Cypress). Repository lacks .github/workflows for automated builds and tests. This increases deployment risk.

🟠 Documentation Gaps HIGH

README contains REPLACE_WITH_PROJECT_ID placeholders. Missing .env.example, CONTRIBUTING.md, LICENSE, and SECURITY.md.

🟡 Sensitive Defaults in Code MEDIUM

Client-side code uses VITE_SUPABASE_PUBLISHABLE_KEY as fallback bearer token. Server-side functions should never expose service-role keys to clients.

✅ Actionable Recommendations

1. 🔐 Rotate Exposed Keys & Remove .env from Repository

  • Immediately revoke and rotate exposed Supabase keys
  • Add .env to .gitignore
  • Create .env.example with placeholder values
  • Consider rewriting git history using git filter-repo

2. 🛡️ Harden Supabase Function Security

  • Enable verify_jwt = true for authenticated functions

3. 📚 Improve Documentation
